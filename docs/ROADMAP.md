# ROADMAP — поэтапная сборка

Каждый этап даёт работающий результат. Можно останавливаться/проверять между этапами.

## Этап 0 — Каркас проекта
- [ ] `create-next-app` (TypeScript, App Router, Tailwind).
- [ ] Установить Prisma + SQLite, инициализировать `schema.prisma`.
- [ ] `.env.example`, базовый layout, шрифты, цвета (тема ЧМ).
- **Результат:** пустой адаптивный сайт с меню (Сегодня / Завтра / Таблица), заглушки страниц.

## Этап 1 — Данные и синк (без UI-красоты)
- [ ] Адаптер `lib/providers/footballData.ts` (запросы к WC: matches, standings).
- [ ] Словарь `teams-ru.ts` (англ → рус для сборных ЧМ-2026).
- [ ] `lib/sync.ts` + роут `/api/sync` — заливка матчей и таблиц в БД.
- [ ] API-роуты: `/api/live`, `/api/matches?date=`, `/api/standings`.
- **Результат:** в БД лежат реальные матчи; роуты отдают JSON. Проверка через браузер/curl.
- ⚠️ Нужен **API-ключ** football-data.org → положить в `.env`.

## Этап 2 — Главный экран «Сейчас в игре»
- [ ] `LiveMatch.tsx`: счёт, минута, статус, команды, флаги.
- [ ] Фон из флагов двух стран (градиент-сплит).
- [ ] Лайв-опрос `/api/live` раз в 60с (SWR/`useEffect`).
- [ ] Fallback, когда live нет (ближайший/последний матч).
- **Результат:** рабочая главная с живым обновлением счёта.

## Этап 3 — Сегодня / Завтра
- [ ] `MatchList.tsx` — список матчей дня (время, команды, флаги, счёт/статус).
- [ ] Страницы `today` и `tomorrow` на базе `/api/matches`.
- **Результат:** расписание на сегодня и завтра.

## Этап 4 — Таблицы
- [ ] `GroupTable.tsx` — таблицы групп (очки, И/В/Н/П, голы, разница).
- [ ] `PlayoffBracket.tsx` — сетка плей-офф (1/16 → финал).
- [ ] Страница `table` с табами «Группы / Плей-офф».
- **Результат:** обе турнирные таблицы.

## Этап 5 — Автосинк и полировка
- [ ] Vercel Cron на `/api/sync` (live — чаще, расписание/таблицы — реже).
- [ ] Адаптивная полировка (мобайл/десктоп), состояния загрузки/ошибок/пустоты.
- [ ] Деплой на Vercel (+ Postgres, если уходим с SQLite).
- **Результат:** боевой сайт, обновляется сам.

## Что нужно от заказчика по ходу
1. **API-ключ** football-data.org (бесплатная регистрация) — перед этапом 1.
2. Подтверждение визуального стиля (цвета/настроение) — перед этапом 2. Можно по референсам.

## Зафиксированные договорённости
См. PROJECT.md (стек, источник данных, язык). Изменения провайдера данных
не ломают фронт — вся внешняя интеграция изолирована в `lib/providers/`.
