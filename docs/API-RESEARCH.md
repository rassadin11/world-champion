# API-RESEARCH — football-data.org

> Документация: https://docs.football-data.org/general/v4/index.html
> Покрытие: https://www.football-data.org/coverage (World Cup входит во free tier)

## Доступ
- Регистрация на football-data.org → бесплатный API-ключ.
- Ключ передаётся в заголовке: `X-Auth-Token: <KEY>`.
- Базовый URL: `https://api.football-data.org/v4`
- **ВАЖНО:** ключ хранить только на сервере (в `.env`, переменная `FOOTBALL_DATA_API_KEY`).
  Никогда не уходит в браузер — все внешние запросы идут через наши API-роуты / cron.

## Лимиты free tier
- **10 запросов в минуту.** → Поэтому опрашиваем внешний API централизованно (cron/фон),
  складываем в свою БД, а клиенты читают из нашей БД. Не по запросу на каждого юзера.
- World Cup доступен. Код турнира: **`WC`**.
- Задержка обновлений на free-тарифе выше, чем на платных. Поминутный «настоящий» live
  не гарантирован, но счёт/статус матча обновляются. Для целей сайта — приемлемо.

## Нужные эндпоинты

| Назначение | Эндпоинт | Примечание |
|---|---|---|
| Все матчи турнира | `GET /v4/competitions/WC/matches` | можно фильтр `?dateFrom=&dateTo=&status=` |
| Матчи по дате (любой турнир) | `GET /v4/matches?date=YYYY-MM-DD` | удобно для «сегодня/завтра» |
| Live-матчи турнира | `GET /v4/competitions/WC/matches?status=LIVE` | live = IN_PLAY + PAUSED |
| Турнирная таблица (группы) | `GET /v4/competitions/WC/standings` | стандинги по группам |
| Один матч | `GET /v4/matches/{id}` | детали конкретного матча |

## Статусы матча (`status`)
`SCHEDULED` · `TIMED` · `IN_PLAY` · `PAUSED` (перерыв) · `FINISHED` ·
`SUSPENDED` · `POSTPONED` · `CANCELLED` · `AWARDED`

«Сейчас в игре» (live) = `IN_PLAY` **или** `PAUSED`.

## Форма ответа по матчу (ключевые поля)
```jsonc
{
  "id": 12345,
  "utcDate": "2026-06-12T19:00:00Z",
  "status": "IN_PLAY",
  "minute": 67,                  // не всегда присутствует на free
  "stage": "GROUP_STAGE",        // или LAST_16, QUARTER_FINALS, ...
  "group": "GROUP_A",            // null на стадии плей-офф
  "homeTeam": { "id": 1, "name": "...", "crest": "https://.../flag.png" },
  "awayTeam": { "id": 2, "name": "...", "crest": "https://.../flag.png" },
  "score": {
    "fullTime": { "home": 1, "away": 0 },
    "halfTime": { "home": 0, "away": 0 }
  }
}
```
- `crest` — URL флага/эмблемы сборной (есть прямо в ответе, отдельный CDN флагов не нужен).
- `stage` различает групповой этап и плей-офф → удобно для двух таблиц.

## Ограничения, которые надо принять
1. **Детальной live-статистики (владение, удары, xG) на free НЕТ.** Показываем: счёт,
   минуту (если есть), статус, забитые голы по таймам. Блок «статистика» строим из того,
   что реально приходит; недостающее не выдумываем.
2. Минута матча (`minute`) приходит не всегда — UI должен это переживать (показывать статус).
3. Названия команд приходят на английском → для русского UI держим маппинг
   код страны → русское название (небольшой словарь, ~48 сборных ЧМ-2026).

## План запасного варианта
Если упрёмся в лимиты или данные неполные — заложить переключение на API-Football
(api-sports.io) без переписывания фронта: вся работа с внешним API изолирована в одном
слое-адаптере (`lib/providers/`), фронт читает только нашу нормализованную модель.
